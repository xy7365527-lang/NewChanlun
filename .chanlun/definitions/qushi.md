## 存在论位置

趋势和盘整是走势类型（Move）的两种**特化子类型**，不是简单的字符串标签。

- **构造层**（第33课）：只有中枢。"盘整、趋势这些概念，其实没有也可以的，唯一有效的就是中枢的概念。"
- **分类层**（第17课）：趋势和盘整是对中枢构造结果的实体化解读，拥有中枢层面不存在的涌现属性。

两层不矛盾——构造层提供基础，分类层提供操作语义。类比：化学中"分子由原子构成"，但水和盐酸是不同物质，有不同性质。

## 定义

### 走势类型三分法 [旧缠论]

> "任何级别的所有走势，都能分解成趋势与盘整两类，而趋势又分为上涨与下跌两类。" — 第17课

> "走势类型三种：上涨、下跌、盘整。上涨、下跌合起来叫趋势。" — 第17课答疑L606

- **二分法**：走势 = 趋势 ∪ 盘整
- **三分法**：走势 = 上涨 ∪ 下跌 ∪ 盘整
- 二分法与三分法中的"盘整"是同一对象（009号定理结论）

### 盘整（Panzheng / Consolidation）[旧缠论]

> "缠中说禅盘整：在任何级别的任何走势中，某完成的走势类型只包含一个缠中说禅走势中枢，就称为该级别的缠中说禅盘整。" — 第17课

**核心特征**：
- 中枢数量：恰好 **1个**
- 方向：**无**（第31课L390："只有趋势才有方向"）
- 完成条件：3个连续次级别走势类型重叠形成中枢后即可完成，但可无限延伸
- 延伸机制：围绕同一中枢 [ZD, ZG] 反复震荡，不产生新中枢
- 背驰类型：**盘整背驰**（力度衰竭不一定终结走势）

### 趋势（Qushi / Trend）[旧缠论]

> "缠中说禅趋势：在任何级别的任何走势中，某完成的走势类型至少包含两个以上依次同向的缠中说禅走势中枢，就称为该级别的缠中说禅趋势。该方向向上就称为上涨，向下就称为下跌。" — 第17课

**核心特征**：
- 中枢数量：**≥2个**（依次同向）
- 方向：**有**（上涨/下跌）
  - 上涨：后DD > 前GG（中心定理二，第20课）
  - 下跌：后GG < 前DD（中心定理二，第20课）
- 完成条件：2个依次同向中枢形成后即可完成，但可无限延续
- 延伸机制：不断产生新的同向中枢
- 背驰类型：**趋势背驰**（必然终结趋势）
- 内部结构：a+A+b+B+c（A/B为中枢，a/b/c为连接段）— 第37课

### 趋势中枢间约束 [旧缠论]

> "缠中说禅走势中枢定理一：在趋势中，连接两个同级别'缠中说禅走势中枢'的必然是次级别以下级别的走势类型。" — 第18课

> "趋势中的中枢之间必须绝对不存在重叠。" — 第18课L28

## 趋势 vs 盘整：差异属性

| 属性 | 趋势 | 盘整 |
|------|------|------|
| 方向 | 有（上涨/下跌）| 无 |
| 中枢数量 | ≥2个（同向）| 恰好1个 |
| 背驰类型 | 趋势背驰（必然终结）| 盘整背驰（不一定终结）|
| 延伸机制 | 产生新同向中枢 | 不产生新中枢 |
| 内部结构 | a+A+b+B+c | 单中枢 + 围绕震荡 |
| 力度度量 | 趋势力度、趋势平均力度 | 无专属度量 |
| 中枢间约束 | 不重叠 | 不适用 |
| 转折后续 | 三种结果（级别扩展/更大盘整/反趋势）| 第三类买卖点 → 趋势或更大盘整 |
| 买卖点关系 | 第一类买卖点=趋势背驰点 | 盘整背驰不产生买卖点 |

## 趋势的生命周期

1. **诞生**：第一中枢后出现第三类买点并形成非背驰类向上（第107课）
2. **形成确认**：至少两个依次同向走势中枢形成（第17/18课定义）
3. **存续/延续**：同向中枢不断产生（第18课）
4. **背驰预警**：a+A+b+B+c 中 c 段力度弱于 a 段（第24/37课）
5. **终结**：趋势背驰确认（第29课背驰-转折定理）
6. **中阴阶段**：趋势终结后到新走势类型确立前的模糊阶段（第88课）

---

## 盘整的生命周期

1. **形成**：三个连续次级别走势类型重叠（第17课）
2. **延伸**：后续 Z走势段与 [ZD, ZG] 有重叠（中心定理一）
3. **终结方式一**：次级别走势离开中枢后回抽不回 → 第三类买卖点（第18课定理三）
4. **终结方式二**：与同级别另一中枢波动区间重叠 → 级别扩张（第20课）

## 当前实现

### 代码状态

- **数据模型**：`Move(kind="trend"/"consolidation")` 字符串标签（`src/newchan/a_move_v1.py`）
- **分类判定**：`len(centers)==1` → consolidation，`len(centers)>=2 && same_direction` → trend
- **方向判定**（v1.1，GG/DD 中心定理二）：
  - `_is_ascending(c1, c2)`: c2.dd > c1.gg → 上涨
  - `_is_descending(c1, c2)`: c2.gg < c1.dd → 下跌

### 声明-实现落差（099号谱系记录）

007/009号决断要求 `Trend(Move)` / `Consolidation(Move)` 子类型，当前实现仍为字符串标签。099号评估：

- kind 字符串足以支撑当前所有分类操作
- 子类型重构在需要趋势力度、编译期背驰类型分离等高级功能时执行
- `qushi.md`（本文件）的创建独立于代码重构——提供可引用的概念锚点

## 边界条件

### 正常情况
1. **最小构成**：任何走势类型至少3段次级别走势（走势分解定理二）
2. **趋势最小构成**：至少2个同向中枢 → 至少5段次级别走势
3. **盘整最小构成**：恰好1个中枢 → 至少3段次级别走势

### 翻转条件
- 当前判定为趋势的走势类型，如果两枢波动区间重叠 → 级别扩张，非趋势
- 当前判定为盘整的走势类型，如果新中枢同向产生 → 升级为趋势
- 如果引入 Trend/Consolidation 子类型 → `Move.kind` 字符串标签方案失效

### 级别扩张（第20课）
- 两个同级别中枢波动区间重叠 → 形成更大级别中枢
- 当前级别走势终结，成为更大级别中枢的一部分
- 判定条件：`(后ZG < 前ZD ∧ 后GG ≥ 前DD) ∨ (后ZD > 前ZG ∧ 后DD ≤ 前GG)`
