# 128号：DAG孤立节点的边建模完整性

**类型**: 矛盾发现
**状态**: 生成态
**日期**: 2026-02-22
**来源**: Gemini异质质询（gemini-rtas-topology-audit.md 焦点2）——16个孤立节点的边建模是否完整

## 发现

v35-swarm RTAS 拓扑审计数据显示：DAG 中有 16 个完全孤立节点（无 depends_on/negates 边）：
001, 002, 003, 004, 008, 011, 023, 024, 035, 098, 099, 107, 108, 109, 113, 114

这些节点在 frontmatter 中有 related 引用，但 related 是弱关联元数据，不参与图的连通性计算。

## 异质质询代理的判定

**判定**：部分成立（Gemini 方向正确，但"16个全部有问题"过于绝对）

逐类分析：
- **合理孤立（根节点/历史记录）**：001（退化段矛盾记录）、002（来源不完整矛盾记录）、003（概念分离记录）、004（元概念框架，其他节点依赖它而非反之）——这些孤立是正确的
- **需要核查**：107（中枢实现选择）、108（背驰实现选择）、109（买卖点实现选择）——这些与主分量的006（级别递归）、007（趋势实体）、009（盘整）有潜在的逻辑依赖，但 DAG 中无边
- **需要核查**：011（定级方向）、023（等价关系）、024（C-2退化）、035（待确认）、098、099、113、114（待逐一检查）

## 张力

- **一侧**：DAG 完整性要求有实质逻辑依赖的节点必须通过 depends_on/negates 建立连接
- **另一侧**：强制给历史矛盾记录和根节点加边会引入假关联，降低 DAG 语义精确性
- **未解决的问题**：107-109 是否真的依赖 006/007/009？需要阅读这三个文件确认逻辑关系

## 推导链

```
Gemini发现: 16个孤立节点 → 声称"拓扑审计完整性声明"与"图谱断裂"冲突
异质质询代理反例: 根节点/历史记录的孤立是正确的（004是所有溯源的根）
部分成立: 需要对107-109等节点专项审查，判断是否存在未建模的依赖边
```

## 待决事项

1. 读取 107、108、109 谱系文件，核查其逻辑依赖
2. 对其余孤立节点（011、023、024、035、098、099、113、114）逐一检查
3. 若发现有实质依赖，在 dag.yaml 中补充 depends_on 边

## 边界条件

- 如果 107-109 的逻辑内容显示它们确实依赖主分量中的节点（如006/007/009），则 negates 稀疏问题加重，边缺失是实质缺口
- 如果 107-109 是独立的实现选择（不依赖领域层定义），则孤立是合理的
