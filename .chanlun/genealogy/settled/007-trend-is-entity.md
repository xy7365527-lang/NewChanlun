## 矛盾描述

趋势在现有实现中仅作为 `Move(kind="trend")` 存在——走势类型实例的一个字符串标签。
编排者指出："趋势和趋势走势类型不一样，不是一个东西"，要求将趋势建模为独立实体对象。

**问题**：原文是否支持趋势作为独立实体？还是仅为走势类型的分类标签？

## 编排者决断（2026-02-17）

**趋势是独立实体对象。建模为 `Trend extends Move` 特化子类型，不是 `Move.kind="trend"` 标签。**

## 趋势独有而盘整不共享的属性/行为

| 属性/行为 | 趋势 | 盘整 |
|-----------|------|------|
| 方向 | 有（上涨/下跌） | 无 |
| 内部结构 | a+A+b+B+c（≥2中枢） | 单中枢，无此结构 |
| 背驰类型 | 趋势背驰（=趋势终结） | 盘整背驰（≠必然终结） |
| 中枢约束 | 不重叠 | 不适用 |
| 延伸机制 | 产生新同向中枢 | 不产生新中枢 |
| 力度度量 | 趋势力度 | 无对应概念 |
| 转折定理 | 三种后续（级别扩展/更大盘整/反趋势） | 第三类买卖点 |

## 趋势的生命周期

1. **诞生**：第一中枢后出现第三类买点并形成非背驰类向上（第107课）
2. **形成确认**：至少两个依次同向走势中枢（第17/18课定义）
3. **存续/延续**：同向中枢不断产生（第18课）
4. **背驰预警**：a+A+b+B+c 中 c 段力度弱于 a 段（第24/37课）
5. **终结**：趋势背驰确认（第29课背驰-转折定理）
6. **中阴阶段**：趋势死亡后到新走势类型确立前的模糊阶段（第88课）

## 与第33课"可省略"声明的关系

第33课L182："盘整、趋势这些概念，其实没有也可以的，唯一有效的就是中枢的概念。"

**不矛盾**。这说的是**构造层**（递归层）只需中枢。但在**分类层**（操作层），趋势和盘整是对构造结果的独立实体化解读，拥有中枢层面不存在的涌现属性（方向、背驰类型等）。类比：化学中"分子由原子构成"，但水和盐酸是不同物质，有不同性质。

## 对代码的影响

1. **当前 `Move(kind="trend")`** → 需要重构为 `Trend(Move)` 子类型
2. **趋势独有属性**（方向、a+A+b+B+c结构、趋势力度）→ 在 Trend 类中定义
3. **盘整** → 对应 `Consolidation(Move)` 子类型
