## 发现过程

007（趋势是实体）、008（趋势确立≠级别确认）、009（盘整双重性）三条谱系的结论存在表面张力：

- 007说趋势是独立实体，有独有属性/行为/生命周期
- 008说"唯一有效的就是中枢"，级别与趋势/盘整无关
- 009说盘整"理论上可省略"但"实践中是实体"

**表面矛盾**：如果"唯一有效的就是中枢"，为什么趋势和盘整又是独立实体？

## 结论（定理）

**缠论体系存在两个不同的抽象层次，趋势/盘整的"理论上可省略"和"实践中是实体"分别属于不同层次：**

### 构造层（递归层）

- **核心对象**：中枢
- **运作方式**：递归函数——三个连续次级别走势类型重叠 → 本级别中枢
- **级别定义**：中枢的递归层级
- **缠师表述**："唯一有效的就是中枢的概念，其他都是多余的"（第33课L182）
- **趋势/盘整在此层**：可省略的推导结果（中枢数量的函数）

### 分类层（操作层）

- **核心对象**：趋势、盘整（走势类型的特化子类型）
- **运作方式**：对构造层产出的实体化解读
- **独立性来源**：涌现属性——方向、背驰类型、力度度量、生命周期等在中枢层面不存在
- **缠师表述**："只有趋势才有方向"（第31课）、"没有趋势没有背驰"（第15课）
- **趋势/盘整在此层**：不可省略的独立实体

### 两层关系

```
分类层：趋势（方向、背驰、力度...） | 盘整（震荡、三类买卖点...）
         ↑ 涌现                        ↑ 涌现
构造层：中枢₁ + 中枢₂ + ...（同向不重叠） | 中枢₁（单枢）
         ↑ 递归                                ↑ 递归
原子层：K线 → 分型 → 笔 → 线段
```

**类比**：化学中"分子由原子构成"（构造层），但"水和盐酸是不同的物质"（分类层）。原子是"唯一有效的"构造单元，但分子拥有原子层不存在的涌现性质（沸点、酸碱性等）。

## 形式化表述

设 `C[k]` 为 k 级中枢，`M[k]` 为 k 级走势类型。

**构造层定义**：
- `C[1]` = 三段 `M[0]`（线段）重叠
- `M[k]` = `C[k]` 的序列 + 连接段
- `C[k+1]` = 三段 `M[k]` 重叠
- 级别 = k

**分类层定义**（从构造层推导）：
- `|centers(M[k])| == 1` → `Consolidation[k]`（盘整）
- `|centers(M[k])| >= 2 ∧ same_direction` → `Trend[k]`（趋势）
- `Trend[k]` 拥有：direction, divergence_type="trend", a+A+b+B+c structure
- `Consolidation[k]` 拥有：divergence_type="consolidation", single_center

**关键**：分类层的属性不是构造层的直接输出，而是对构造层结果的**解读**产生的涌现属性。

## 对代码架构的影响

1. **构造层**（RecursiveStack, RecursiveLevelEngine）：不需要知道趋势/盘整——只处理中枢和走势类型
2. **分类层**（Trend, Consolidation 子类型）：基于构造层产出进行实体化——赋予方向、背驰类型等属性
3. **定级别**（determine_level）：纯构造层操作——基于中枢判断，不涉及分类
