## 发现过程

2026-02-17 ceremony 中，蜂群三工位（趋势实体性/级别独立性/盘整身份）完成后，工作流程出现了错误：

**错误路径**：蜂群完成 → 汇总结论给编排者 → 编排者确认 → 补写谱系
**正确路径**：蜂群完成 → **立即写谱系** → 谱系之间的张力暴露新洞察 → 再汇总

编排者指出："看谱系和写谱系非常重要，通过谱系也许你就能找到答案了。"

## 实证

### 错误路径的损失

- 007（趋势是实体）和 008（级别靠中枢不靠趋势）和 009（盘整"理论上可省略"）表面有张力
- 如果先写谱系，写007的"谱系链接"时就必须面对"这和第33课说可省略怎么办？"
- 写008的"谱系链接"时就必须面对"如果级别只靠中枢，趋势的独立性在哪里体现？"
- 这两个追问碰撞出来的就是**010号谱系：构造层vs分类层的二层架构**

### 正确路径的收益

- 010不是"三个结论的汇总"，而是"三个张力的解决方案"
- 这个解决方案只在**写谱系的过程中**才会被发现，因为写谱系迫使你：
  1. 写清推导链（暴露逻辑缺口）
  2. 写清谱系链接（暴露概念间张力）
  3. 写清前置/关联（暴露依赖关系）

## 规则（语法记录）

### 谱系写作时机

**每个蜂群工位完成后，在汇总结论之前，先写对应的谱系记录。**

不是"搜索完了再统一补记"，而是"每完成一个概念性发现，立刻写入谱系"。

### 理由

谱系的六个字段（状态、推导链、谱系链接、影响、来源、类型）构成了一个**强制思考框架**：
- "推导链"迫使你检查逻辑是否严密
- "谱系链接"迫使你检查新发现与旧发现是否一致
- 如果不一致，就是一个新的矛盾——需要新的谱系记录来解决

**谱系是矛盾的显现机器，不是产出的记录日志。**

### 蜂群 + 谱系的正确工作流

```
蜂群工位A完成 → 立刻写谱系A → 检查谱系A与已有谱系的张力
蜂群工位B完成 → 立刻写谱系B → 检查谱系B与A和已有谱系的张力
蜂群工位C完成 → 立刻写谱系C → 检查谱系C与A/B和已有谱系的张力
                                ↓
                         张力暴露 → 新谱系D（解决张力）
                                ↓
                         汇总报告给编排者
```
