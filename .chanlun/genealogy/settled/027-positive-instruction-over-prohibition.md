# 027 — 正面指令优于禁止性规则

**状态**: 已结算
**日期**: 2026-02-18
**结算日期**: 2026-02-18
**来源**: 编排者观察 + 实践验证
**溯源**: [新缠论]
**四分法**: 语法记录（已在运作但未显式化的规则）

## 发现

禁止性规则（"commit 后不要停"）对抗语言模型的生成惯性，失败率高。
正面指令（"commit 后下一个输出必须是行动声明 + tool 调用"）顺着生成惯性，成功率高。

## 推导链

1. **观察**: 规则 7 写了"commit/push 不是断点"，但 agent 反复在 commit 后输出总结段落并隐式等待
2. **诊断**: commit 完成后 LLM 自然要生成一段文本——禁止性规则要求压制这个冲动，但压制自身的生成惯性是 LLM 最难做到的事
3. **解法**: 不压制惯性，而是重定向惯性——规定这段必然会生成的文本的格式为行动声明，然后紧跟 tool 调用
4. **推论**: 惯性内容从"总结 → 等待"变成"声明 → 执行"，系统行为不变但成功率提升

## 一般性原则

对 LLM agent 的行为规则，**正面默认动作 > 禁止性约束**。

- 不说"不要做 X"，说"做 Y"（Y 是 X 的替代）
- 不说"不要输出总结"，说"输出行动声明"
- 不压制生成惯性，重定向生成惯性

## 影响
