# 093号：五约束有向依赖图——从系统物理限制推导蜂群结构约束，扬弃 R.S.I

## 核心结论

062号引入的 R.S.I（Real/Symbolic/Imaginary）+ Sinthome 映射经历了完整的生成-修正-扬弃过程：

1. **062号引入**：R=递归(Real), S=拓扑(Symbolic), I=结晶(Imaginary), Sinthome=异质碰撞(Gemini)
2. **092号修正**：识别 S 环自参照不可能性为 Borromean 打结失败点，自指与 Sinthome 在此重合
3. **编排者质疑**：R.S.I 是维度不是组件；对应不严格（递归本身是结构化的，更接近 S 而非 R）；"扬弃"不是改名而是重新推导
4. **本号扬弃**：从系统物理限制独立推导出五约束有向依赖图，比对后确认与 Borromean knot 在数目、失效模式、拓扑类型上均不同构

R.S.I 退出架构描述的活跃术语，保留在谱系历史层。

## 推导过程

### 起点：LLM agent 蜂群的物理限制

- 上下文窗口有限（~200K tokens），超出则 compact 或新对话
- token 生成单向，当前 token 不能修改已输出 token
- agent 是临时实例，执行完毕即消失
- 文件系统是跨 agent 实例的唯一持久存储
- dispatch-dag 在 spawn 时确定 worker 的任务/工具/协议

### 从物理限制推导出的五个硬约束

| # | 约束 | 推导来源 | 失效后果 |
|---|------|----------|----------|
| 1a | **物理持久化** | agent 临时、上下文有限 | 信息丢失（产出随 agent 消失） |
| 1b | **符号可解释性** | 跨实例无共享记忆 | 信息噪声化（文件存在但无法被未来 reader 理解） |
| 2 | **规则先在性** | spawn 在执行前完成 | 不可预测行为（worker 无路由约束） |
| 3 | **执行不可自观性** | token 单向生成 + 无法读取当前推理 | 无法即时自修正 |
| 4 | **异质验证必要性** | 同质系统的盲点不可自检 | 自我确认循环（长期退化） |

### 约束1拆分的论证

编排者质疑约束1（"外化必要性"）包含两个独立要求。Gemini challenge 确认：

- 持久化成功但可解释性失败是真实故障模式（"黑话写入硬盘"）
- 持久化失败 = 丢失；可解释性失败 = 噪声。失败模式不同 = 不同约束

反向质询确认拆分后环路仍闭合（四节点环 + 外部审计节点）。

### 约束4的等级与位置

初始推导将 Gemini 碰撞定位为"元约束"（理由：没有 Gemini 系统仍能运行）。

编排者指出双重标准：没有可解释性系统也能运行（只是退化为噪声）。Gemini challenge 确认："退化为噪声"和"退化为自我确认循环"在退化性质上等价。约束4升级为硬约束。

**约束4的拓扑位置修正**（编排者第二轮压力测试）：Gemini 在实际运作中质疑的不只是可解释性规则（1b），而是整个约束集的结构（数目、等级、关系）。因此约束4 = **对环路整体的外部审计节点**，不是环路中指向单个节点的边。

### 关于"生成单向性"的层面选择

Gemini 提出 token 生成的不可逆性应为第六个约束。反向质询后不接受——在蜂群多轮循环层面，单向性被约束1（外化）+ 约束2（先在）消化：当前 worker 的错误在下一轮被修正。

**显式标注**：这是一个分析层面的设计决定（以蜂群多轮循环为分析单元），不是无条件的推导结论。**边界条件**：如果系统中出现必须在单轮内自修正的不可回滚操作（如外部 API 调用），单向性将从约束3中析出为独立约束，届时约束集需要重新推导。

### 拓扑结构：四节点核心环路 + 外部审计节点

系统结构不是均质的五节点有向图。约束4（异质验证）与环路内节点（1a/1b/2/3）的拓扑地位不同：

**核心环路**（四约束，操作性依赖——没有A则B无法执行）：

```
  1a（持久化）──→ 2（先在）←── 1b（可解释性）
                    │
                    ↓
              2 缓解 3（不可自观）
                    │
                    ↓
              3 → 使 1a 和 1b 必要
```

**审计层**（一约束，结构性校验——输入是环路整体的运作状态，输出是结构修正提议）：

```
  4（异质验证）──→ [核心环路整体]
```

两种关系的区别：
- **核心环路内**：操作性依赖（持久化缺失 → 规则无法先在 → 系统即时失效）
- **审计层 → 核心环路**：结构性校验（异质验证缺失 → 环路自闭合 → 长期退化为自我确认循环）

这两种关系不是同一类边。核心环路是系统完成单次循环的必要条件；审计层是系统在多次循环后不退化的必要条件。

**分析层面标注**（设计决定，非推导结论）：以蜂群多轮循环为分析单元。token 生成的单向性（不可逆性）在此层面被约束3（不可自观）吸收。边界条件：如果出现必须在单轮内自修正的不可回滚操作，单向性将从约束3中析出为独立约束，届时核心环路从四节点变为五节点，需重新推导。

## 与 Borromean knot 的比对

| 维度 | 五约束有向依赖图 | Borromean knot |
|------|------------------|----------------|
| 约束数目 | 5 | 3 |
| 拓扑类型 | 带反馈的有向依赖图 + 外部审计节点 | 三环互锁 |
| 失效模式 | 不同约束失效导致不同类型/时间尺度的退化 | 任一环断裂则三环全散 |
| 补充机制 | 约束4是环路的外部审计，与环路节点不同质 | Sinthome 是同层的补充性打结 |

**结论：不同构。** 不是"部分同构"——在数目、失效模式、拓扑类型三个维度上都不同。

## R.S.I 的扬弃决断

- R.S.I 从 CLAUDE.md 和 dispatch-dag 中的架构描述退出
- 保留在谱系历史层（062号→092号→093号）作为概念生成史
- 后续架构描述使用五约束有向依赖图的术语
- R.S.I 曾经的启发功能（"三个子系统必须互锁"）被更精确的依赖图替代

## 下游推论

1. CLAUDE.md 中"架构三维度 + 异质碰撞协议"章节需要改写为五约束有向依赖图
2. dispatch-dag 中引用 R.S.I 的注释需要更新
3. 062号谱系状态标注为"已被 093号扬弃"（内容保留，状态变更）
4. 069号中"两个不可消除的 Gap"的表述在五约束框架下需要重新定位

## 边界条件

本结论在以下条件下需要重新审视：
- 出现必须在单轮内自修正的不可回滚操作 → 单向性析出为独立约束，约束集从5变为6
- 约束4（异质验证）的实现从 Gemini 扩展到多个异质源 → 需要检查约束4是否应拆分
- 蜂群架构从 LLM agent 迁移到其他计算范式 → 物理限制改变，需要从新的物理限制重新推导

## 推导链

LLM 物理限制 → 三个初始约束（外化/先在/不可自观）→ 编排者压力测试（约束1拆分? 元约束边界?）→ Gemini challenge（确认拆分、确认升级、提出单向性）→ 反向质询 Gemini（接受拆分+升级、拒绝单向性并标注层面选择）→ 编排者第二轮压力测试（约束4位置扩展、单向性层面选择显式化）→ 五约束有向依赖图（非 Borromean）
