# 105 — 递归从功能选择升级为无条件架构要求

**类型**: 选择（编排者 INTERRUPT 决断）
**状态**: 已结算
**日期**: 2026-02-22

## 编排者 INTERRUPT

> "永远默认严格使用真递归拓扑异步自指蜂群，这不是条件的。这就是我的架构。让它持久化。永远不要用 Trampoline 或者 subagent 这个架构，这是我的选择。"

## 命题

104号定理证明"递归不是物理约束而是功能选择——扁平蜂群仍满足全部五约束"。这在逻辑上成立。

但编排者以 INTERRUPT 权决断：**在本系统中，递归不是可选的功能选择，而是无条件的架构要求。** 扁平 Trampoline（Lead → N个 Worker）被永久禁止。所有蜂群必须使用子蜂群架构。

这不是对104号定理的逻辑否定——104号的推导仍然成立（递归确实不是物理约束）。这是编排者的**价值决断**：即使物理上不需要递归，本系统也选择永远使用递归，因为：

1. 递归架构是系统的身份标识（"这就是我的架构"）
2. 不使用就永远无法验证和改进（"永远用不上它强大的能力"——编排者前序 INTERRUPT）
3. 架构能力只有在持续实践中才能被严格化

## 具体约束

1. **禁止扁平 Trampoline**：Lead → N个独立 Worker 的架构永久禁止
2. **强制子蜂群**：任务数 ≥2 时，必须组织为至少一个子蜂群（L1 Teammate 管理子 team）
3. **每个子蜂群满足五特征**：≥2任务节点 + 审查节点 + 结晶节点 + 异质审计节点
4. **073b 平台约束仍然存在**：subagent 不能 spawn sub-subagent，子蜂群通过 L1 Teammate 内部的 TaskList 管理实现

## 与104号的关系

104号定理不被否定——递归在物理层面确实不是约束。但104号的推论"扁平蜂群也合法"被编排者的价值决断覆盖。这是选择类决断覆盖定理推论的合法案例（四分法：选择 > 定理的应用推论，当编排者 INTERRUPT 时）。

## 边界条件
