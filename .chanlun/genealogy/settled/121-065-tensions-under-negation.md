# 121号：065号张力边在068否定后的存废——被否定框架下的张力自动失效

**类型**: 定理
**状态**: 已结算
**日期**: 2026-02-22
**来源**: v32审计发现 T6 张力——068否定065后，065的 tensions_with 边残留

## 张力描述

065号（六条未解决张力）使用连续流形语言描述了拓扑连续性 vs 断裂性的统一矛盾。068号（范式转换）否定了连续流形假设，将建模语言转为偏序集/有向图。

065号自身已被068号否定（065.negated_by = ["068"]，065头部有历时性拓扑补丁标注）。但065号在 DAG 中仍有 tensions_with 边（与062号和064号），这些边在068号否定后是否应该清理？

## 推导链

```
前提1: 068号否定065号（dag.yaml negates: from 068 to 065）
前提2: 065号的 tensions_with 边描述的张力基于连续流形假设
  - 065↔062: Sinthome 与连续流形撕裂的张力
  - 065↔064: 异质对话与连续流形假设的张力
前提3: 068号否定的正是连续流形假设
前提4: 张力的存在依赖其表述框架——框架被否定后，张力的表述变为语法不合法
────────────────────────────────────────────────────────────
结论: 065号的 tensions_with 边在068号否定后自动失效。
       但作为历时性记录应保留在 DAG 中（不删除），标注为"在已否定框架下"。
```

## 结论

**065号的 tensions_with 边在068号否定后自动失效，但不应删除。**

理由：
1. **失效**：065号使用"拓扑手术"、"流形撕裂"等连续流形语言描述张力。068号否定了这些语言的合法性。在离散图论框架下，这些张力需要重新表述（如果仍然存在）才能恢复为活跃张力。
2. **不删除**：谱系必须维护（原则4）。tensions_with 边记录的是历史上曾存在的张力，删除它们会丧失历时性信息。正确做法是在 DAG 中标注这些边的状态。
