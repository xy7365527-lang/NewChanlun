# 123号：真中枢/假中枢概念的谱系入口——递归链完整性的构造层判据

**类型**: 语法记录（已在运作的概念显式化入DAG）
**状态**: 已结算
**日期**: 2026-02-22

## 概念描述

真中枢（Authentic Center）和假中枢（Apparent Center）是 level_recursion.md 中引入的 [新缠论] 概念对，用于判断递归链的完整性：

- **真中枢**：通过完整递归链构造的中枢——从底层K线出发，经过包含→分型→笔→线段→中枢的完整递归。每一层都是从下层严格构造而来。
- **假中枢**：直接在高级别K线上识别的中枢——跳过了中间递归层。形式上可能满足"三段重叠"，但缺少递归链支撑。

## 概念的来源与理由

缠论原文使用级别递归定义中枢（第17-21课），但未显式区分"通过递归构造的中枢"和"直接在高级别K线上识别的中枢"。这个区分在多周期下钻方法中不存在（因为各周期独立处理），只在严格级别递归框架下才有意义。

006号决断选择了级别递归（而非多周期下钻）后，真/假中枢的区分成为必要：
- 如果使用多周期下钻 → 每个周期独立构造中枢，无"真/假"问题
- 如果使用级别递归 → 中枢必须从下层递归构造，未经递归的中枢是"假"的

因此真/假中枢是006号决断的下游推论——选择了级别递归就必须区分递归链完整性。

## 推导链

```
前提1: 006号决断——级别 = 递归层级（level_id），不是时间周期
前提2: 中枢定义依赖三段重叠（缠师原文第17-21课）
前提3: "三段"的"段"必须是次级别走势——递归链中每层的"段"都是下层构造的产物
────────────────────────────────────────────────────────────
推论: 未经递归链构造的"中枢"（直接在高级别K线上识别）不满足定义中的"次级别走势"要求
结论: 需要区分真中枢（递归链完整）和假中枢（递归链不完整）
```

## 在代码中的运作

level_recursion.md 定义了递归终止条件：当数据不足以在当前级别构造完整的笔/线段时，递归终止。终止层之上的中枢如果强行构造，就是假中枢。

代码中通过 `RecursiveOrchestrator` 的递归深度控制来避免假中枢的生成。

## 边界条件

- 如果006号决断被推翻（改用多周期下钻）→ 真/假中枢概念失去意义
- 如果递归实现允许跳层构造（如从1分钟直接跳到日线中枢）→ 需要重新定义"假中枢"的边界

## 下游推论

1. 真/假中枢的区分是006号的推论，不是独立的设计决策
2. 任何涉及高级别中枢构造的代码都应检查递归链完整性
