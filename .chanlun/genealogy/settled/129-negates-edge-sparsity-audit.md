# 129号：negates边稀疏性审查

**类型**: 定理（三轮Gemini双向严格讨论消解）
**状态**: 已结算
**日期**: 2026-02-22
**来源**: Gemini异质质询（焦点5）→ 三轮双向严格讨论

## 问题

DAG 拓扑中 negates 边仅8条，vs depends_on 311条、tensions_with 20条。Gemini 判定：系统存在"矛盾降级"——将实质否定关系软化为张力。

## 推导链

```
数据: 8条 negates, 20条 tensions_with, 311条 depends_on
Gemini 假设: 稀疏性 = 矛盾降级（和稀泥）

实地检查（Lead 同步执行）:
1. 20条 tensions_with 逐一检查——无一条存在逻辑互斥
   - 007↔008/009, 008↔009: 概念互补张力，非互斥
   - 020↔005b: 127号已消解（层面不同）
   - 062↔065/069/094/104: 062已被否定（negated_by: 064, 093），张力边是历史残留
   - 其余: 从不同角度切入同一问题域的概念张力
2. 8条 negates 全部是真正的取消/替代关系
3. 9个"矛盾记录"类型节点——矛盾通过概念分离消解，分离产出新节点不产出 negates 边
4. 概念分离机制（005→005a/005b等）是 negates 稀疏的合理解释

Gemini 三轮讨论:
- 第一轮: "和稀泥"判定·致命
- 第二轮: 给出两个实例，均有逻辑缺陷（循环论证 + 已否定节点）
- 第三轮: 放弃提供具体互斥实例，转向方法论批评（边类型设计缺失）
─────────────────────
结论: "矛盾降级"指控不成立——无具体互斥实例支撑。
negates 稀疏是系统概念分离机制的正常结果，不是和稀泥。
```

## 结论

**negates 边的稀疏性（8/311）不构成系统缺陷。**

1. negates 语义 = "A被B取消/替代"（强否定），tensions_with = "A和B有未解决张力"。两者语义不同，不可直接比较。
2. 系统中矛盾的主要消解方式是概念分离（产出新节点），不是直接否定（产出 negates 边）。
3. 20条 tensions_with 边逐一检查，无一条应升级为 negates。
4. Gemini 三轮讨论未能提供任何不依赖焦点3（127号判定）的互斥实例。

## Gemini 的方法论建议（有价值但不改变判定）

Gemini 第三轮提出：系统在"逻辑未决"时默认使用 tensions_with，而不是 needs_review——这是方法论层面的合理批评。建议：
- 考虑引入 `needs_review` 边类型，区分"确认的持续张力"和"未决的待分类关系"
- 这是分类学卫生改进，属于建议级别，不影响当前系统功能
