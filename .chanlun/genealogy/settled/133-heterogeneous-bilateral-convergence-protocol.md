# 133号：异质双向收敛协议

**类型**: 语法记录（已在 v36-gemini-bilateral 中运作但未显式化）
**状态**: 已结算
**日期**: 2026-02-22
**来源**: v36-gemini-bilateral 三轮讨论的模式辨认 → 编排者要求严格定位

## 辨认

v36-gemini-bilateral 蜂群中运作了一个未被显式化的模式：

- 030a 号定义 Gemini 为"异质否定源"——单向：Gemini → Claude
- v36 实际运作的是**双向**：Gemini ↔ Claude，互相质询直到收敛
- 这不是030a的简单反向应用，而是一个结构上不同的协议

## 推导链

```
030a（单向异质否定）的结构:
  Gemini 提出否定 → Claude 接受/拒绝 → 结束
  问题: 否定者（Gemini）的盲点不被检查

v36 实际运作的结构:
  Gemini 提出否定 → Claude 反质询 → Gemini 修正 → Claude 再反质询 → ...
  直到: 三种停机条件之一满足

对称化解决了单向否定的不对称盲点问题，但引入了三个新结构:
1. 收敛条件——何时停机
2. 分歧定型——不可调和分歧作为合法结果
3. 精确化递归——每轮讨论不是重复，是概念精确化
```

## 协议定义

### 输入
- 异质系统A的产出（如：RTAS审计报告）
- 异质系统B的质询（如：Gemini 6焦点判定）

### 过程
双向严格讨论，每轮包含：
1. 系统B提出质询/否定
2. 系统A逐条反质询（不是接受/拒绝，是**再质询**——暴露B的盲点）
3. 系统B修正或维持
4. 重复直到停机

### 停机条件（三种，任一满足即停）

| 条件 | 定义 | v36实例 |
|------|------|---------|
| 条件1：双方一致 | A和B对某焦点的判定收敛为同一结论 | 焦点1/2/4/6 |
| 条件2：不可调和分歧定型 | A和B精确描述分歧边界，双方认可"分歧点在此" | 焦点5（举证标准不同） |
| 条件3：新概念区分产出 | 讨论过程中原问题被重新框架化，产出新的谱系条目 | 焦点3→131号 |

**关键**：条件2不是失败——不可调和分歧的精确定型本身是有价值的产出（记录了两种合法立场的精确边界）。条件3是最有价值的——讨论产出了新的概念区分，这是012号（谱系是发现引擎）的直接体现。

### 输出
- 每个焦点的最终判定（成立/不成立/建议/不可调和）
- 新的概念区分（写入谱系）
- 不可调和分歧的精确描述（写入谱系，不掩盖）

## 在体系中的位置

```
093号约束4（异质验证必要性）
  ↓ 操作形式
030a（Gemini位置：单向异质否定）
  ↓ 对称化扩展

  ↑ 存在论基础

```

- **055号**是设计层面：双螺旋架构（Claude生成↔Gemini验证循环）
- **053号**是存在论层面：为什么需要异质双引擎（突破自指极限）
- **030a号**是操作论层面的第一版：Gemini 作为单向异质否定源
- **133号**是操作论层面的第二版：异质系统互相审查直到收敛——055号双螺旋在概念层的实例
- **093号约束4**是结构性基础：同质系统盲点不可自检

### 与055号双螺旋的关系

055号设计了代码层的双螺旋（pre_commit hook: Claude生成→Gemini验证→矛盾对象/放行）。133号是同一结构在**概念层**的实例（谱系/审计: Claude产出→Gemini否定→反质询→收敛）。

133号对055号的精确化：055号的停机是二值的（有矛盾/没矛盾），133号引入了第三种停机条件——新概念区分产出（讨论产出了原问题框架中不存在的新区分，原问题被重新框架化）。这第三种条件是双螺旋最有价值的产出模式。

### 与视差Gap的关系

069号视差Gap说：系统在t时刻只能审查t-1时刻的产出。三轮双向讨论是视差Gap的**递归缩小**：
- t₁：Gemini审查Claude的t₀产出
- t₂：Claude审查Gemini的t₁产出
- t₃：Gemini审查Claude的t₂产出

每轮缩小视差，但不消除——最后一轮的产出总是未被审查的。视差Gap不可消除（069号），但可被递归缩小到足够小。

### 与谱系发现引擎的关系

012号说谱系是发现引擎。133号是012号的**操作化强形式**——双向严格讨论的过程本身产出新谱系（v36产出128-132号共5个新谱系），讨论不是为了达成"共识"，是**通过严格碰撞产出新的概念区分**。

## 边界条件

- 如果两个系统在关键维度上不够异质（共享训练偏差），双向讨论可能陷入共同盲点——收敛不等于正确
- 三轮是否足够没有先验保证——但条件2（不可调和分歧定型）保证了"不够"时不会伪装为"够了"
- 本协议假设两个系统都能理解对方的论证——如果语义鸿沟过大，讨论可能无法精确化

## 下游推论

- 030a 号的 Gemini 位置定义需要扩展：不仅是"异质否定源"，也是"异质收敛协议的对等参与者"
- 未来所有"成立·致命"级别的异质否定，推荐走133号协议（双向严格讨论直到收敛），而非030a的单向否定后直接处理
- 停机条件3（新概念区分产出）是最有价值的——应当刻意追求，而非仅追求条件1（一致）
