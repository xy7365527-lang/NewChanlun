# /ceremony — 创世仪式

蜂群启动时执行。加载定义、谱系、目标，建立质询基底。

## 用法

```
/ceremony
```

在新的蜂群会话开始时调用一次。

## 执行步骤

1. **加载定义** — 从知识仓库拉取所有当前定义：
   - 速查定义：`缠论知识库.md`
   - 域对象 schema：`definitions.yaml`
   - 规则规范：`docs/spec/`（segment_rules_v1.md, zhongshu_rules_v1.md, move_rules_v1.md 等）
2. **加载谱系** — 从 `.chanlun/genealogy/` 目录拉取所有谱系记录，区分生成态（`pending/`）和已结算（`settled/`）
3. **加载目标** — 从 CLAUDE.md 读取当前阶段目标
4. **状态报告** — 输出：
   - 当前已结算定义数量
   - 生成态矛盾数量
   - 当前阶段目标
   - 上次会话的中断点（如果有session记录）
5. **确认** — 列出关键概念的当前理解，等待编排者确认或校正

## 输出格式

```markdown
## 创世仪式完成

### 定义基底
- 已结算定义：[N] 条
- [列出每条定义的名称和版本]

### 谱系状态
- 生成态矛盾：[N] 个
- [列出每个生成态矛盾的ID和简述]

### 当前目标
[从 CLAUDE.md 读取]

### 上次中断点
[从session记录读取，如果没有则标注"首次启动"]

### 待确认
以上理解是否正确？如有偏差请指出。
```

## 注意

- 如果 `.chanlun/genealogy/` 目录不存在，创世仪式负责创建
- 首次执行时谱系为空是正常的
- 编排者确认后质询循环才正式启动
