# /escalate — 矛盾上浮

当质询过程中发现不可弥合的概念矛盾时，生成标准格式的矛盾报告呈交编排者。

## 用法

```
/escalate
```

在质询过程中发现矛盾后调用。Claude会根据当前上下文自动生成报告。

## 报告格式

```markdown
## 矛盾报告

### 矛盾描述
[什么跟什么冲突，为什么不可弥合]

### 双方论证
**立场A：** [agent X的结论及其定义依据]
**立场B：** [agent Y的结论及其定义依据（或质询中发现的不一致）]

### 涉及的定义
[列出相关定义条目，引用知识仓库（缠论知识库.md / definitions.yaml / docs/spec/）]

### 谱系比对结果
[是否有历史先例？如果有，上次怎么解决的，为什么这次不同]

### 需要决断的问题
[精确地说明编排者需要判断什么——不是"这里有问题"，
而是"A和B哪个应该成立，或者是否需要概念分离"]
```

## 注意

- 「需要决断的问题」必须用领域语言（缠论术语），不是代码语言
- 提供足够信息让编排者做判断，但不替编排者做判断
- 报告生成后自动保存到 `.chanlun/genealogy/pending/` 目录
