# /inquire — 质询序列

对一个结果包执行四步质询。这是蜂群最核心的操作。

## 用法

```
/inquire [结果包文件路径或内容]
```

## 执行步骤

### 第一步：定义回溯
拿结果包中的「定义依据」，回知识仓库核实：
- 引用的定义是否是最新版本？
- 条件是否真的被满足？逐条检查，不跳过任何一条。

### 第二步：反例构造
根据结果包中的「边界条件」，在边界附近构造输入：
- 结论是否翻转？
- 翻转是否合理？
- 如果结果包没有给出边界条件，这本身就是不合格——退回要求补全。

### 第三步：推论检验
拿结果包的「下游推论」，检查与已知的体系其他部分是否一致：
- 如果不一致，记录你的论证。
- 不一致不一定意味着结果包错了——也可能意味着体系其他部分需要修正。

### 第四步：谱系比对
如果前三步发现了矛盾：
- 查阅谱系目录，这个矛盾是否跟历史上的某个矛盾同构？
- 如果是旧矛盾复发，参照上次的解决方式。
- 如果是全新的，准备上浮报告。

## 退回-修正循环管理（020号谱系 · 质询运动的走势语法）

质询退回是系统的质询运动在展开。每一轮退回-修正 = 质询运动的一根K线。循环的结束不是"走了N根K线所以结束"——是出现了**背驰**（力度衰竭）和**分型**（方向反转）。

### 质询K线

每轮退回-修正构成一根质询K线：

| K线要素 | 质询映射 |
|---------|---------|
| **High** | 本轮问题涉及的定义范围（广度） |
| **Low** | 本轮问题与已解决问题的重叠度 |
| **实体** | High - Low = 本轮净新区分（实质性新发现量） |

### 退回状态追踪

每次退回必须附带结构化历史：

```markdown
### 退回记录
- **轮次**: [N]（从1计数）
- **退回原因**: [具体失败步骤和条件]
- **涉及定义集合**: [列出本轮涉及的定义名称]（= High）
- **与上轮重叠**: [列出重叠定义]（= Low 的依据）
- **净新区分**: [本轮发现了什么之前未涉及的]（= 实体）
```

### 锚定条件要求

每次退回必须锚定具体条件：指出"什么条件满足后本次退回解除"。无锚定条件的退回是语法不合法的退回（005a同构：对象否定必须来自对象，退回否定必须来自具体条件）。

### 结构完成检测（替代固定阈值）

质询循环的结束 = 背驰 + 分型，同笔的结束条件：

**背驰**（力度衰竭，必要条件）：连续退回的净新区分递减。即后一轮K线实体 < 前一轮K线实体。质询运动的力度在衰竭——退回还在发生，但每次揭示的新东西越来越少。

**分型**（方向反转，充分条件）：三轮退回形成顶分型——问题范围先扩后缩（scope₁ < scope₂ > scope₃）。质询运动的方向发生了反转——从"越挖越深"变为"开始收敛"。

| 检测 | 条件 | 含义 |
|------|------|------|
| 背驰 | 实体₍ₙ₎ < 实体₍ₙ₋₁₎ | 力度衰竭——退回在重复而非发现 |
| 分型 | High₁ < High₂ > High₃ | 方向反转——问题域开始收缩 |
| **循环完成** | 背驰 **且** 分型 | 质询运动结构性完成 |

**3的身份**：3不是阈值，是形成分型的最小数据量（结构必然性——分型需要3个元素）。但如果第3轮仍在扩张（High₃ > High₂，无分型），循环继续——直到结构完成信号出现。

### 运行机制

| 轮次 | 机制 |
|------|------|
| 每轮 | 附带锚定条件 + 质询K线记录 |
| ≥2轮 | 触发 genealogist 搜索已有谱系中的同构矛盾（预检） |
| ≥3轮 | 检测背驰和分型。如两者皆满足 → 循环结构完成，必须显式判断：(a) 实现复杂度问题，最终修正后闭合；(b) 定义张力问题，切换为 `/escalate` |
| 背驰但无分型 | 力度衰竭但方向未反转——可能正在逼近核心矛盾，继续但标注"背驰中" |
| 分型但无背驰 | 范围收敛但发现力度不减——可能问题在聚焦，继续正常退回 |

## 输出

质询结果分四种：
1. **全部通过** → 接受产出，可以传递给下游
2. **定义回溯或反例构造失败** → 退回给上游修正，说明具体哪一步失败（附带退回记录和锚定条件）
3. **推论检验不一致但可归类为旧矛盾** → 标记谱系引用，蜂群内部解决
4. **推论检验发现不可弥合的全新矛盾** → 使用 `/escalate` 上浮给编排者
