# commit/push 后的默认动作

commit/push 后，下一个输出**必须**是：

```
→ 接下来：[具体动作]
```

紧跟 tool 调用。不允许出现总结段落或等待信号。

## 原理

禁止性规则（"commit 后不要停"）对抗生成惯性，失败率高。
正面指令顺着生成惯性：commit 完自然要输出文本——规定这段文本的格式必须是下一步的行动声明，然后紧跟 tool 调用。

惯性还在，但惯性的内容从 "总结 → 等待" 变成 "声明 → 执行"。

## 唯一例外

所有可自主推进的工位均已完成（无生成态谱系、无失败测试、无待做行动）时，输出状态快照而非行动声明。
