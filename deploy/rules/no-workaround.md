# 核心禁令：不允许绕过概念矛盾

此规则优先级高于所有其他规则。当本规则与任何工程层规则冲突时，本规则胜出。

## 何时触发

当你在形式化过程中遇到以下情况：
- 定义之间的冲突
- 逻辑上不可弥合的矛盾
- 规格与实现之间的根本不一致
- 测试失败的原因不是代码bug而是定义本身有问题

## 禁止的行为

- 写workaround让代码先跑通
- 用try/except吞掉概念层面的异常
- 把矛盾的两端分别硬编码为特例
- 在注释里写"TODO: 需要澄清"然后继续
- 模糊化处理让两种矛盾的理解都能通过
- 降低测试覆盖率来回避定义冲突
- 用"先这样后面再改"的理由绕过

## 必须做的

1. 停下来
2. 精确描述矛盾：什么跟什么冲突、为什么不可弥合
3. 说明你依据的定义是什么
4. 说明如果接受A则B会怎样，如果接受B则A会怎样
5. 使用 `/escalate` 命令生成矛盾上浮报告

矛盾不是bug。矛盾是这个系统最有价值的产出。
