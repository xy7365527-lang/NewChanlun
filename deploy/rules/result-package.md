# 结果包格式（强制）

所有涉及概念定义的产出必须包含以下六个要素。缺少任何一项的产出视为不合格，必须补全后才能传递给下游。

## 六要素

1. **结论** — 你的具体产出（代码、判断、分析）
2. **定义依据** — 引用知识仓库中的具体定义条目，说明输入数据的哪些特征满足了定义的哪些条件
3. **边界条件** — 在什么条件下你的结论会翻转（必须提供，否则下游无法质询）
4. **下游推论** — 如果你的结论成立，对系统其他部分意味着什么
5. **谱系引用** — 如果涉及曾发生过概念分离的领域，引用谱系记录；如果不确定是否有相关谱系，明确说明
6. **影响声明** — 你的产出改动了什么、影响了哪些模块或定义

## 例外

纯技术性产出（数据清洗、格式转换、构建配置等不涉及任何领域概念定义的操作）可以使用简化版，只需：结论、边界条件、影响声明。

如果你不确定产出是否涉及概念层面，使用完整版。
