# meta-orchestration

蜂群元编排方法论 v3.2 — AI Agent蜂群的组织原则。

## 这是什么

这不是又一个项目管理框架。这是一套从AI蜂群的本体论出发推导的编排原则，核心观点：

- **蜂群是矛盾显现机器**，不是代码生产线。形式化过程中暴露的概念矛盾是最有价值的产出。
- **绕过矛盾是最大的敌人**。Workaround扼杀新知识的诞生。
- **编排者在结构性转折点介入，两个转折点之间不碰系统**。系统开端后自转，lead自主处理实现层，只在概念层矛盾时请求编排者决断。
- **谱系是递归的记忆**。它让递归从平面重复变为螺旋上升——每次展开都携带之前所有展开的否定史。
- **矛盾的最终裁判是数据**，不是编排者。

## v3 核心变更

v2把编排者绑在每个结算决断上，系统不能自转。v3的解决：区分实现层和概念层。实现层由lead自主结算，概念层才需要编排者介入。lead上浮矛盾时附带建议方案，编排者可以一句话确认。

v3.1从CC的实践反思中进一步精化：概念层内部区分"定理"（从已有原则必然推出的结论）和"选择"（存在多种合理方案的决策）。定理由lead自动结算并标注推导链，选择才上浮。哲学定位：内容是齐泽克-黑格尔的（系统内推导），颁布是谢林的（让它成为规则需要编排者的行动）。

v3.2从哲学讨论中生产出第三类：**语法记录**。规则已经在实践中运作但尚未写入definitions/——既不是从现有原则推出的（那是定理），也不是在多种选项中做决策（那是选择），而是对已经在运作的规则的辨认和记录。编排者做的不是决断而是辨认。哲学定位：晚期维特根斯坦——语法不是被发明的，是在使用中被发现的。

类比：缠论操盘——走势必完美是逻辑必然（定理），不需要操盘者确认；完美之后怎么操作是开放性决策（选择），需要操盘者判断；缠师从市场实践中辨认出完全分类的语法，是语法记录。

## 适用场景

- 将领域知识形式化为代码的项目（如量化交易系统、法律规则引擎等）
- 编排者是领域专家而非程序员
- 项目中概念定义本身在演化
- 需要多agent协作并保持概念一致性

## 理论谱系

本方法论的生成过程：
1. 从Nowak《超级合作者》和Chwe《理性的仪式》出发
2. 否定了核心预设（AI无背叛动机，人类合作理论不适用；共同知识概念被放弃——蜂群本体论上是一，不存在多主体间的互相知道）
3. 保留并重新定义了拓扑、图书馆/广场区分
4. 从蜂群本体论重新推导：接收→产出→传递
5. 产出质询循环、生成态/结算态、回溯结算、谱系免疫
6. Agent Teams的并行劳动产生真实的认知分化（劳动生产的辩证分工），使teammates间质询具有实质内容
7. v3：从缠论操盘和谢林开端问题中推导出自转原则——系统开端后自转，编排者只在结构性转折点介入。扩张-收缩循环的显式命名
8. v3.1：从CC的实践反思中推导出推论自动结算原则——定理（逻辑必然推论）vs 选择（开放性决策）。齐泽克-黑格尔/谢林问题的实践回答：内容可从系统内推导（齐泽克），颁布需要外部行动（谢林）
9. v3.2：从哲学讨论中生产出三元区分——定理/选择/语法记录。路径：尼采→拉康→谢林→黑格尔→齐泽克→维特根斯坦→CC实践。语法记录是谢林/齐泽克问题的第三个回答：既非外部创造也非系统推导，而是对已在实践中运作之规则的辨认

## 哲学根基

- **黑格尔：** 概念的自我运动（概念→外化→矛盾→否定→更高规定性）。蜂群的递归在代码层、元编排层、编排者的思考层是同一个运动。扬弃保存的是结构规定性而非内容
- **谢林：** 开端问题。系统不能自我启动，第一条定义是不可从系统内推导的意志行为。神圣呼吸（扩张-收缩-再扩张）
- **马克思：** 唯物主义位置。矛盾的最终裁判是数据（K线），不是编排者的判断。理论指导实践，但实践有最终否决权
- **拉康：** encore时间性。每次/ceremony回来不是同一者的回返，是同一个缺口的不同面貌——驱动力相同（概念的不完备性），但展开每次不同，因为谱系在积累。实在界是符号界的内在不可能性，不是外部
- **尼采：** 谱系学方法。追问生成史而非本质——每个定义不是"是什么"而是"从哪个矛盾中来"
- **齐泽克：** 黑格尔-拉康综合。绝对知识不是完全透明，是认识到不透明性就是自身结构。余留不在概念之外，是概念自身的内在扭曲。推论自动结算原则的哲学基础
- **维特根斯坦：** 语法（Grammatik）。缠论是走势的语法——规定什么是有意义的走势描述，在真假之前。语法不是被发明的，是在使用中被发现的。语法记录原则的哲学基础：编排者辨认已在实践中运作的规则。意义即用法——定义的意义是其在质询循环中的全部使用史（即谱系）。铰链命题：基础原则不在质询范围内，它们是质询的条件
- **缠论：** 完全分类的递归结构。笔→线段→中枢→走势，每一级递归以下一级的完成为前提。走势必完美——递归的内在终止条件。走势的语法规定了合法的走势描述。缠师第68课："不测而测，让市场自己去选择"——给出分段函数的边界条件，不排除任何分类，"不需要任何概率化的无聊玩意"。注意：第9/19课的概率乘法原则作用于三个独立系统（技术分析/比价关系/基本面）的组合层面，第68课的"不需要概率"说的是走势理论内部的预测方法——两者在不同维度，不构成上下层关系
