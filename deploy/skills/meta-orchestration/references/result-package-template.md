# 结果包模板

每一个agent产出都必须使用此格式。裸结果（只给代码或结论）不可接受。

```markdown
## 结果包

### 结论
[你的具体产出——代码、判断、分析]

### 定义依据
- 依据定义：[引用知识仓库中的具体条目]
- 条件满足说明：
  - 条件1：[定义要求X] → [输入数据的Y特征满足此条件，因为...]
  - 条件2：[定义要求X] → [满足/不满足，说明...]
  - ...

### 边界条件
- 当 [条件A] 时，结论翻转为 [结论B]
- 当 [条件C] 时，结论不确定
- 已知的边界区域：[描述]

### 下游推论
- 如果本结论成立，则 [模块/定义X] 应当 [推论1]
- 如果本结论成立，则 [模块/定义Y] 不应当 [推论2]
- 潜在冲突区域：[如果你预感某个推论可能与体系其他部分矛盾，标注出来]

### 谱系引用
- 相关谱系记录：[GEN-XXX]（如果涉及曾经发生过概念分离的领域）
- 无相关谱系 / 不确定是否有相关谱系（明确说明）

### 影响声明
- 改动的文件/模块：[列表]
- 影响的下游模块：[列表]
- 影响的定义：[列表]
```

## 简化版（用于低风险的纯技术产出）

如果产出是纯技术性的（如数据清洗、格式转换）且不涉及任何概念定义，可以使用简化版：

```markdown
## 结果包（简化）

### 结论
[产出]

### 定义依据
无概念定义涉及，纯技术操作。

### 边界条件
[输入数据的什么特征会导致处理失败]

### 影响声明
[改动了什么]
```

但如果你不确定产出是否涉及概念层面，使用完整版。宁可多写不可少写。
